# Spotify Clone - Database Management System Version Lite

This database project was developed as a final assignment for the Database Systems course (Tugas Besar Mata Kuliah Basis Data). The project focuses on designing and implementing a robust database architecture for a music streaming platform similar to Spotify, ensuring data integrity, relational consistency, and transaction reliability.

## 🛠 Tech Stack
- **DBMS:** MySQL / MariaDB
- **Storage Engine:** InnoDB (Supports ACID transactions)

## 📂 Repository Structure
- `/src/schema.sql`: Complete database structure, primary keys, foreign keys, and constraints.
- `/queries/transactions.sql`: Examples of `START TRANSACTION`, `COMMIT`, and `ROLLBACK` for data integrity.
- `/queries/reports.sql`: Analytic queries using `JOIN`, `GROUP BY`, and aggregation functions.

## 🚀 How to Run
1. Create a new database: `CREATE DATABASE sistem_dbspotify;`
2. Import `src/sistem_dbspotify.sql`.
3. Run operational queries from the `/queries/` folder.
